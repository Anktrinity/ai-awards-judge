# GitHub Publish Checklist

## Before pushing

- [x] Separate public-safe assets from private entrant data
- [x] Add license
- [x] Add README
- [x] Add docs for methodology, deployment, and hybrid publishing
- [x] Add config-driven scaffold
- [x] Add sanitized examples
- [x] Initialize local git history
- [ ] Authenticate GitHub CLI or set remote manually
- [ ] Create remote repository
- [ ] Push `main`

- Visibility: Public
- Issues: On
- Wiki: Off for now
- Discussions: Optional
- Default branch: `main`
- Add topics: `ai`, `awards`, `judging`, `evaluation`, `event-tech`, `open-source`
